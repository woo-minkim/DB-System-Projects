# Project 2: 부동산 거래 DB 구현 및 질의 프로그램

Project 2는 Project 1에서 설계한 모델을 실제 관계형 DB(MySQL)로 구현하고,  
C++ 기반 콘솔 프로그램에서 질의/집계/데이터 입력 기능을 수행하는 과제입니다.

## 과제 목표

- 모델링 결과를 실제 물리 스키마로 구현
- 샘플 데이터 적재 후 질의 시나리오 검증
- 메뉴 기반 인터페이스를 통해 조회 및 입력 기능 실행
- 과제 요구 TYPE 질의(1~7) 구현

## 실행 흐름

1. 프로그램 시작 시 MySQL 서버 연결 및 `project` DB 선택
2. `20212019_1.txt`를 읽어 테이블/샘플 데이터 생성
3. 콘솔 메뉴에서 TYPE 1~7 질의 수행
4. 종료 시 `20212019_2.txt` 실행으로 객체 정리

## TYPE 질의 기능 요약

### TYPE 1: Mapo 매물 조회

- 기본: 주소에 `Mapo`가 포함된 매물 주소 조회
- 서브쿼리: `listing_price`가 10억~15억 범위인 Mapo 매물 조회

### TYPE 2: 학군/방 개수 조건 조회

- 기본: 학군(`school_district`)이 `8`인 매물 주소 조회
- 서브쿼리: 학군 `8` + 침실 4개 이상 + 욕실 2개 조건 조회

### TYPE 3: 에이전트 실적 집계

- 기본: 2022년 판매 총액 기준 상위 에이전트 조회
- 서브쿼리 1: 2023년 판매 총액 상위 `K`명 조회
- 서브쿼리 2: 2021년 판매 총액 하위 10% 에이전트 조회

### TYPE 4: 가격/시장 체류 기간 분석

- 기본: 2022년 에이전트별 평균 판매가격 + 평균 시장 체류기간(`sale_date - listed_date`)
- 서브쿼리 1: 2023년 에이전트별 최고 판매가
- 서브쿼리 2: 에이전트별 최대 시장 체류기간

### TYPE 5: 유형별 최고가 매물 이미지 조회

- `studio`, `one_bedroom`, `multi_bedroom`, `detached_house`별 최고가 매물의 `image_url` 조회

### TYPE 6: 거래 데이터 입력

- 사용자 입력으로 `Transaction`에 신규 거래 삽입
- 연결 테이블 `Buy`에도 관련 데이터 삽입

### TYPE 7: 에이전트 데이터 입력

- 사용자 입력으로 `Agent` 신규 등록

## 입력/출력 예시 형식

```text
------- SELECT QUERY TYPES -------
1. TYPE 1
...
7. TYPE 7
0. QUIT
```

- TYPE 3-1 입력 예: `K` 값을 입력하면 상위 K명 결과 출력
- TYPE 6 입력 예: 거래 ID, 매물 ID, 매수자 ID, 매도/매수 에이전트 ID, 가격, 날짜를 순서대로 입력
- 출력 형식: 결과 행을 공백 구분 텍스트로 콘솔 출력

## 내가 한 일

- ER 모델 기반으로 테이블/참조관계 DDL을 설계하고 SQL 스크립트화했습니다.
- 샘플 데이터를 직접 구성해 조인/집계 질의 검증이 가능하도록 데이터셋을 준비했습니다.
- C++에서 SQL 실행 함수(`executeSQLQuery`)와 결과 출력 함수(`printQueryResults`)를 분리해 질의 재사용성을 높였습니다.
- TYPE 1~7 요구 기능을 메뉴형 워크플로우로 구현해 조회/분석/입력을 한 프로그램에서 처리하도록 만들었습니다.
- 종료 시 정리 스크립트를 분리해 실습 반복 실행 시 환경 복구가 가능하도록 구성했습니다.

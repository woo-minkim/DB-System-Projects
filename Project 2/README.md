# Project 2. Real Estate Transaction DB System

부동산 거래 도메인을 **MySQL 물리 스키마로 구현**하고  
이를 **C++ 콘솔 프로그램과 연동해 조회·집계·입력 기능까지 수행하도록 만든 DB 응용 프로젝트**입니다.

Project 1에서 설계한 ER 모델을 실제 실행 가능한 데이터베이스 시스템으로 확장했으며  
조인·서브쿼리·집계 질의를 통해 데이터 활용 시나리오를 검증했습니다.

## Key Highlights

- ER 모델을 기반으로 관계형 물리 스키마 구현
- 샘플 데이터 적재 및 질의 시나리오 검증
- C++에서 MySQL 연동 및 메뉴 기반 콘솔 인터페이스 구현
- 조회, 집계, 데이터 입력 기능을 하나의 프로그램으로 통합

## Tech Stack

- **Language**: C++
- **Database**: MySQL
- **Modeling**: ERwin
- **Interface**: Console-based menu system

## Main Features

- 조건 기반 매물 조회
- 학군, 방 개수, 가격 조건을 반영한 검색 질의
- 연도별 에이전트 판매 실적 집계 및 순위 분석
- 평균 판매가 및 시장 체류 기간 분석
- 매물 유형별 최고가 이미지 URL 조회
- 신규 거래 및 에이전트 데이터 입력

## My Contributions

- Project 1의 ER 모델을 실제 MySQL 테이블 구조와 참조관계로 구현했습니다.
- PK/FK 제약을 반영한 DDL을 작성하고 샘플 데이터를 구성해 조인 및 집계 질의를 검증했습니다.
- SQL 실행 함수와 결과 출력 함수를 분리해 질의 로직의 재사용성과 가독성을 높였습니다.
- TYPE 1~7 기능을 메뉴형 워크플로우로 구현해 조회·분석·입력 기능을 한 프로그램 안에서 수행하도록 구성했습니다.
- 초기화 SQL과 종료용 정리 SQL을 분리해 반복 실행과 테스트가 가능하도록 설계했습니다.

## Execution

1. MySQL에 `project` 데이터베이스를 생성합니다.
2. `20212019.cpp` 내 DB 접속 정보를 로컬 환경에 맞게 수정합니다.
3. 프로그램 실행 시 초기 SQL이 적용되고 메뉴를 통해 기능을 수행할 수 있습니다.
4. 종료 시 정리 SQL을 실행해 생성 객체를 제거합니다.
